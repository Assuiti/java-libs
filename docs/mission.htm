Mission Statement of the
openEHR Java Reference Implementation Project

Purpose

The purpose of this document is to state the missions of the Java Reference Implementation Project for people to understand what the Java project is aiming for and what it can provide.

Missions

  1. Validate published design specifications through implementation activities and provide concrete feedbacks that will hopefully lead to improvement of the specifications.
  2. Create an implementation in Java that is both faithful to the design specifications and technically sound thus provide educational purpose towards prospective openEHR developers.
  3. Explore different implementation strategies and try to find most optimal solutions for given deployment requirement.
  4. Provide common code base grouped as coherent, reusable software components as building blocks for full-fledged EHR systems.
